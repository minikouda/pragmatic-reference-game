\section{Project Overview}
% -----------------------------------------------------------------------

This project investigates \emph{when} a language model agent should ask a clarifying
question versus commit to an answer in a \textbf{reference game} setting.  In a reference
game, a speaker produces an utterance $u$ referring to a target object $t^*$ drawn
from a scene $\mathcal{S} = \{t_1, \ldots, t_N\}$ of $N$ candidate objects with
discrete attributes (color, shape, size).  The listener must identify $t^*$.

The core insight is that clarification should be treated as a
\emph{cost-utility optimisation}: asking a question incurs a fixed cost $c$, but
resolves ambiguity; committing directly risks an incorrect answer when the utterance
is ambiguous.  We use the \textbf{Rational Speech Act} (RSA) framework to derive a
posterior $P(t_i \mid u)$ and then make the decision that maximises expected utility:
\begin{equation}
    \text{decide to ask} \iff
    \underbrace{1 - c}_{E[U_{\text{ask}}]} >
    \underbrace{\max_i\, P(t_i \mid u)}_{E[U_{\text{commit}}]}.
\end{equation}

\section{Phase 1: Metric Definitions (Shizhe)}
% -----------------------------------------------------------------------

\subsection{Brier Score (multi-class adaptation)}

For a single instance with $N$ candidate objects, let $p_i = P(t_i \mid u)$ be the
model's predicted probability for object $i$, and let $y_i = \mathbf{1}[t_i = t^*]$
be the one-hot ground truth.  The multi-class Brier Score is
\begin{equation}
    \mathrm{BS} = \frac{1}{N}\sum_{i=1}^{N}(p_i - y_i)^2.
\end{equation}
Averaged over $M$ instances: $\overline{\mathrm{BS}} = \frac{1}{M}\sum_{j=1}^{M}\mathrm{BS}^{(j)}$.

\textbf{Range and baselines.}
$\mathrm{BS} = 0$ when all probability mass is on $t^*$; a uniform listener scores
$\mathrm{BS}_{\text{uniform}} = (N-1)/N^2 \approx 1/N$.

\textbf{Worked example.}
Scene: \{red square, blue circle, red triangle\}, utterance: ``the red one.''

\begin{center}
\begin{tabular}{lccc}
\toprule
Object & $p_i$ & $y_i$ & $(p_i - y_i)^2$ \\
\midrule
red square   & 0.50 & 1 & 0.25 \\
blue circle  & 0.10 & 0 & 0.01 \\
red triangle & 0.40 & 0 & 0.16 \\
\midrule
\multicolumn{3}{r}{$\mathrm{BS}$} & $0.14$ \\
\bottomrule
\end{tabular}
\end{center}

\textbf{Why it matters.}
Brier score measures whether the \emph{full} distribution $P(t \mid u)$ is accurate.
Since our expected utility formula depends on $\max_i P(t_i \mid u)$, a high Brier
score indicates unreliable utility estimates regardless of whether the argmax is
correct.

\subsection{Expected Calibration Error (ECE)}

ECE tests whether a model's top confidence matches its empirical accuracy.

\textbf{Procedure.}
\begin{enumerate}
    \item For each instance $j$, extract $\mathrm{conf}_j = \max_i P(t_i \mid u)$ and
          $\mathrm{correct}_j = \mathbf{1}[\arg\max_i p_i = t^*]$.
    \item Partition instances into $B$ equal-width bins on $[0,1]$ (default $B = 10$).
    \item For each non-empty bin $b$ compute $\mathrm{acc}(b)$ and $\overline{\mathrm{conf}}(b)$.
    \item
    \begin{equation}
        \mathrm{ECE} = \sum_{b=1}^{B} \frac{|b|}{M}\,
        \bigl|\mathrm{acc}(b) - \overline{\mathrm{conf}}(b)\bigr|.
    \end{equation}
\end{enumerate}

\textbf{Why it matters.}
If ECE is large, the model is systematically over- or under-confident.  Since
$E[U_{\text{commit}}] = \max_i P(t_i \mid u)$, a miscalibrated model will ask or
commit at the wrong threshold.  ECE $> 0.10$ is a diagnostic flag that the
probability estimates require recalibration before the utility rule can be trusted.

\section{Phase 1 Experiment}
% -----------------------------------------------------------------------

\subsection{Setup}

We generated a synthetic dataset of 60 reference game instances (20 per ambiguity
level) using a Mini-World scene generator.  Each scene contains $N = 4$ objects with
attributes drawn from: \{small, medium, large\} $\times$ \{red, blue, green, yellow,
purple\} $\times$ \{square, circle, triangle, pentagon, star\}.

Ambiguity level is controlled by how much of the target's attribute description
appears in the utterance:
\begin{itemize}
    \item \textbf{Low}: utterance specifies all three attributes (e.g., ``large red square'') --- unique referent.
    \item \textbf{Medium}: utterance specifies only colour (e.g., ``the red one'') --- 1--2 matching objects.
    \item \textbf{High}: utterance specifies only size (e.g., ``the large object'') --- all objects match.
\end{itemize}

Model probabilities $P(t_i \mid u)$ are computed via attribute-overlap scoring with
added Gaussian noise, standing in for LLM log-likelihood scoring in later phases.

\subsection{Discussion}

The experiment validates three core claims of the project:

\begin{enumerate}
    \item \textbf{Entropy stratifies ambiguity correctly.}
    Low-ambiguity scenes concentrate near $H = 0.5$; high-ambiguity scenes cluster
    near $\ln 4 = 1.39$ (Figure~\ref{fig:entropy_brier}).

    \item \textbf{The cost-utility rule makes sensible decisions.}
    At $c = 0.25$, the agent never asks in low-ambiguity scenes and asks 100\% of the
    time in high-ambiguity scenes (Table~\ref{tab:summary}).  The clarification rate
    curves (Figure~\ref{fig:clarification}, left) monotonically decrease as $c$
    increases, as expected.

    \item \textbf{ECE is borderline but acceptable.}
    Overall ECE $= 0.105$ (Figure~\ref{fig:ece}).  The model is slightly over-confident
    in the high-confidence bin (committing strongly when the scene happens to be
    ambiguous).  This motivates RSA-based recalibration in Phase 3.
\end{enumerate}

\section{Phase 2: VLM Speakers and Listeners}
% -----------------------------------------------------------------------

\subsection{Speaker Design}

\textbf{Target identification.}
VLM speakers receive the plain scene image plus the target object's properties
injected as text in the system prompt:
\texttt{"the SIZE COLOR SHAPE at LOCATION (scene position: x=X, y=Y where 0,0
is bottom-left and 100,100 is top-right)"}.
The exact $(x,y)$ coordinates are provided for spatial grounding (so the model
can reason about ``leftmost'', ``top-right'', etc.) but the model is explicitly
instructed \emph{not} to output raw coordinates in its response --- only natural language.
No image annotation (bounding boxes, overlays) is used; the model must locate
the target purely from the textual description paired with its own visual perception.

\textbf{Speaker strategies.}
We evaluate seven prompting strategies for \texttt{StrategicVLLMSpeaker}:

\begin{itemize}
    \item \textbf{natural} --- Unconstrained; produce a short referring expression
          that uniquely identifies the target.
    \item \textbf{contrastive} --- Find the most confusable distractor and
          highlight the single distinguishing feature.
    \item \textbf{landmark} --- Describe the target's position relative to a
          visually distinctive nearby object.
    \item \textbf{superlative} --- Use an extreme or uniqueness property
          (``the only triangle'', ``the largest circle'', ``the leftmost blue shape'').
    \item \textbf{pragmatic} --- RSA-style chain-of-thought: list all objects,
          identify distractors, produce the minimal discriminating expression.
    \item \textbf{scene\_first} --- Inventory all visible objects first, confirm
          the target, then produce the shortest unique expression.  Mitigates
          hallucination by grounding description in an explicit object inventory.
    \item \textbf{listener\_aware} --- Explicitly reason from the listener's
          perspective: produce the description that would maximise the listener's
          confidence in the correct object, making every other object clearly wrong.
    \item \textbf{canonical\_vllm} --- Constrained vocabulary strategy: the model
          must choose COLOR, SHAPE, SIZE, and LOCATION from fixed closed sets
          (e.g.\ shape $\in$ \{circle, square, triangle, star, pentagon, hexagon\}),
          include features greedily until the description is unique, and output
          only a canonical ``the [SIZE] COLOR SHAPE [at LOCATION]'' expression.
          Designed to match the utterance format of the rule-based
          \textit{feature-canonical} baseline.
\end{itemize}

Four rule-based baselines (no LLM) are included: \textit{ordinal}
(superlatives/uniqueness), \textit{landmark} (spatial reference),
\textit{contrastive} (foil-based contrast), and \textit{feature-canonical}
(greedy minimal canonical feature description).  The \textit{feature-canonical}
speaker is the strongest rule-based baseline and the primary comparison target.

\subsection{Listener Design}

All listeners receive the scene image, a numbered text list of every object's
attributes, and the speaker's utterance.  They output a posterior
$P(t_i \mid u)$ over $N$ objects.  Five variants were implemented:

\begin{itemize}
    \item \textbf{vllm-listener} ($\sigma{=}10$) --- Predicts $(x,y)$ coordinates;
          posterior is a Gaussian kernel $\exp(-d_i^2/2\sigma^2)$.
    \item \textbf{feature-match} --- Extracts a feature tuple from the utterance
          and scores each object by feature-overlap count.
    \item \textbf{direct-rank} --- Outputs a probability array $[p_0,\ldots,p_{N-1}]$
          directly; softmax-normalised.
    \item \textbf{cot-rank} --- Assigns a score $s_i \in [0,10]$ to each object
          before converting to probabilities.
    \item \textbf{elimination} --- Explicitly lists \textsc{ruled\_out} indices
          (probability~0), then scores the remaining \textsc{candidates}.
\end{itemize}

\textbf{Posterior flatness fix.}
The initial \texttt{vllm-listener} used an inverse-distance kernel
$\text{score}_i = 1/(d_i{+}\epsilon)$, giving mean $\max(P){=}0.52$ for
$N{=}6$ objects and a near-100\% ask rate at $c{\leq}0.25$.
Replacing it with $\exp(-d_i^2/2\sigma^2)$, $\sigma{=}10$ canvas units raises
the mean max-posterior to 0.83 and the commit rate at $c{=}0.25$ from 4\% to 72\%,
with no change to the argmax prediction.

\subsection{Experiment 2: Diagnosing the Rule-Based vs.\ VLM Gap}
\label{sec:exp2}

\textbf{Root cause analysis.}
Comparing \textit{feature-canonical} and \textit{strategic-natural} utterances
on the same scenes reveals three systematic VLM failure modes:

\begin{enumerate}
    \item \textbf{Vague shape vocabulary} (20\% of VLM utterances): the model outputs
          ``the black shape'' or ``the black blob'' instead of the specific shape name.
          Feature-canonical always names the exact shape; VLMs use generic words
          when uncertain.
    \item \textbf{Wrong attribute}: the model hallucinates a property --- e.g.\
          ``the black circle'' when the target is a black square, or ``the red square''
          when the target is a red triangle.
    \item \textbf{Under-specification}: the model omits a needed discriminating
          feature --- e.g.\ ``the green square'' when size is required to distinguish
          two green squares of different sizes.
\end{enumerate}

Quantitatively: feature-canonical includes the exact shape name in 84\% of utterances
and color in 100\%, with zero vague words.  Strategic-natural includes the shape name
in only 80\% and produces vague generic words in 20\%.

\textbf{The \texttt{canonical\_vllm} strategy.}
Motivated by this analysis, we introduced a new prompting strategy that constrains
the VLM speaker to use closed vocabularies (shape $\in$ \{circle, square, triangle,
star, pentagon, hexagon\}; color, size, location analogously) and follow the same
greedy feature-inclusion logic as the rule-based speaker: include shape+color first,
add size only if ambiguous, add location only if still ambiguous.
The model is explicitly told it must not use synonyms or vague words outside the
specified vocabulary.

Table~\ref{tab:canonical_gap} shows results on \texttt{gemini\_exp\_test.jsonl}
after this fix, with the bounding-box annotation removed (target provided as text only).

Notably, \textit{listener\_aware} (0.92 CPA) approaches rule-based performance via
a different route: listener-centric reasoning naturally produces discriminative
descriptions without requiring explicit vocabulary constraints.

\paragraph{Key findings.}

\textbf{The listener is the bottleneck on harder scenes.}
With a strong \textit{direct-rank(claude-sonnet-4)} listener, \emph{all} speakers
--- naive or engineered, small or large --- achieve CPA $\geq 0.94$.  The speaker
strategy becomes secondary.  By contrast, pairing any speaker with
\textit{elimination(gemini-flash)} drops CPA to 0.78--0.79, indicating that the
elimination listener's higher ask rate (4--8\%) is the dominant cost driver.

\textbf{Annotation failure mode resolved.}
In earlier experiments (with bounding-box annotation), claude-sonnet-4 produced
utterances describing the annotation itself (``the magenta square'', ``the highlighted
object'') rather than the actual object, yielding CPA~${\approx}0.12$.
After switching to text-only target identification, claude-sonnet-4 as a
\emph{listener} becomes the top performer.  This finding motivated the design
change from visual annotation to textual target passing in \texttt{StrategicVLLMSpeaker}.

\textbf{Prompt engineering narrows but does not eliminate the performance gap.}
On the harder 6-object dataset, engineered gemini-flash speakers
(\textit{listener\_aware}: 0.95--0.97 CPA) closely match naive large-model speakers
(0.95--0.98).  The gap between them is within experimental noise at $n{=}50$.
The earlier advantage of prompt engineering observed in Experiment~1 (0.54 vs.\ 0.30)
was partly attributable to the bounding-box annotation failure harming large models
disproportionately.

\section{Experiment 4: Full-Scale Evaluation Across Scene Complexities}
\label{sec:fullscale}
% -----------------------------------------------------------------------

\subsection{Setup}

To assess how speaker and listener strategies generalise beyond the small
pilot datasets used in Experiments~1--3, we conducted a full-scale evaluation
spanning six scene configurations that vary both the number of objects and the
degree of forced perceptual overlap between distractors.

\textbf{Datasets.}  All six datasets were generated with our synthetic scene
renderer (\S\ref{sec:exp1}):
\begin{itemize}
    \item $N \in \{6, 8, 10\}$ objects per scene,
    \item \textit{none}: objects drawn independently (low to moderate ambiguity),
    \item \textit{force}: at least one distractor shares the target's color
          (guaranteed near-confusable pair, high ambiguity),
    \item $n = 300$ scenes per configuration (1\,800 scenes total).
\end{itemize}

\textbf{Systems evaluated.}
\begin{itemize}
    \item \textbf{Speakers:} \textit{natural}, \textit{listener\_aware},
          \textit{canonical\_vllm} (all using gemini-2.0-flash-001), and the
          rule-based \textit{feature-canonical} oracle.
    \item \textbf{Listeners:} \textit{direct-rank} and \textit{elimination},
          both using gemini-2.0-flash-001.
    \item \textbf{Cost sweep:} $c \in \{0.10, 0.25, 0.50\}$.
\end{itemize}

All VLM speaker calls provide the target's properties as structured text
(color, shape, size, location, and exact $(x,y)$ coordinates) with an explicit
instruction not to output raw coordinates.  No image annotation is used.

\subsection{Analysis}

\paragraph{Finding 1: \textit{direct-rank} is uniformly superior to \textit{elimination}.}
Across all 24 (speaker, dataset) pairs, \textit{direct-rank} outperforms
\textit{elimination} by an average of $\Delta\text{CPA} = 0.106$ at $c{=}0.5$.
The gap widens with scene complexity: at $N{=}6$, the advantage is $\approx 0.06$;
at $N{=}10$, it grows to $\approx 0.20$.
This degradation of the elimination listener is explained by its ask rate:
at $N{=}10$, \textit{elimination} asks 20--27\% of the time, whereas
\textit{direct-rank} asks only 1--4\%.
The elimination strategy's tendency to rule out objects aggressively (and
then be uncertain among the remainder) is well-suited to small, unambiguous scenes
but becomes overconservative as the candidate set grows.

\paragraph{Finding 2: \textit{canonical\_vllm} is the best VLM speaker across all settings.}
Among VLM speakers, \textit{canonical\_vllm} achieves the highest CPA in 9 of 12
(dataset, listener) conditions.
Averaged over all six datasets with \textit{direct-rank}, it achieves mean
CPA $= 0.966$, versus $0.952$ for \textit{listener\_aware} and $0.941$ for
\textit{natural}.
The vocabulary constraint --- forcing the model to name shape, color, size, and
location from closed sets --- eliminates vague generic words (``the black shape'')
and attribute hallucinations that are the primary failure modes of unconstrained VLM speakers.

\paragraph{Finding 3: Scaling from $N{=}6$ to $N{=}10$ has modest impact on \textit{direct-rank}.}
With \textit{direct-rank}, mean CPA across speakers decreases by only $0.021$
from $N{=}6$ to $N{=}10$ (none variant: $0.948 \to 0.973 \to 0.973$; note
non-monotone due to variation in scene difficulty).
This robustness suggests that the direct-ranking listener effectively marginalises
over larger candidate sets without systematic degradation.

\paragraph{Finding 4: Forced overlap (\textit{force}) has negligible impact with \textit{direct-rank}.}
Comparing \textit{none} vs.\ \textit{force} variants at the same $N$, CPA differences
are $\leq 0.015$ for \textit{direct-rank} across all speakers.
The forced same-color distractor, which was designed to increase ambiguity and
force speakers to rely on additional discriminating features (shape, size, location),
does not substantially degrade performance.
This suggests that the VLM listener can resolve same-color ambiguity from the
image and structured object descriptions without special accommodation from
the speaker.

\paragraph{Finding 5: The residual gap between \textit{canonical\_vllm} and \textit{feature-canonical} is consistent.}
Feature-canonical outperforms canonical\_vllm by $\Delta\text{CPA} \approx 0.026$
on average with \textit{direct-rank}.
This gap is entirely attributable to \emph{under-specification}: even with a
closed vocabulary, the VLM occasionally omits a needed discriminating feature
(e.g.\ outputting ``the blue triangle'' when size is required).
The rule-based speaker eliminates this by construction --- it greedily adds
features until the description is provably unique with respect to the ground-truth
object set.  Closing this remaining gap would require either (a) giving the VLM
speaker the full distractor list and asking it to verify uniqueness, or (b)
a post-hoc uniqueness check that adds the missing feature programmatically.

\paragraph{Summary.}
The best \emph{fully-VLM} pipeline is
\textit{canonical\_vllm(gemini-flash)} + \textit{direct-rank(gemini-flash)},
achieving CPA $= 0.947$--$0.990$ across all six datasets at $c{=}0.5$.
This represents a substantial improvement over the initial VLM baseline
(CPA $\approx 0.43$ at the start of Phase~2) and approaches the rule-based
oracle ceiling (CPA $= 0.982$--$0.997$) to within $\approx 0.03$ CPA points.
The primary remaining bottleneck is VLM speaker under-specification, not
listener error.

\subsection{Motivation: The Leakage Problem}

Experiments~1--4 used text-assisted listeners: every listener prompt included a
numbered list of the form
\begin{center}
\texttt{[0] small red circle at bottom-left}\\
\texttt{[1] large blue square at top-right}\\
\texttt{\ldots}
\end{center}
alongside the scene image and the speaker's utterance.
While this provided a controlled testbed for measuring speaker quality, it
introduced a \emph{feature leakage} problem: a listener that simply matches the
utterance tokens against the pre-formatted object list can achieve near-perfect
accuracy without ever examining the image.
The CPA scores reported in Table~\ref{tab:fullscale} (0.94--0.997) are therefore
an overestimate of genuine visuo-linguistic understanding.

Concretely, given the speaker utterance ``the large blue circle'' and the formatted
list above, a string-matching procedure suffices: the text immediately identifies
object~[1] if it were ``large blue circle''.  The listener's visual channel is
entirely bypassed.

A fully general listener should operate under the same informational constraints
as a human participant in a real reference game: \emph{it sees the scene image and
hears the utterance, and nothing more}.  We call such a listener
\textbf{image-only} (IO), in contrast to the text-assisted (TA) baselines of
Experiments~1--4.

\subsection{Image-Only Listener Design}
\label{sec:io-design}

\paragraph{Index annotation.}
To give IO listeners a shared referential vocabulary with the speaker, we render
each scene image with integer index labels (0, 1, \ldots, $N{-}1$) placed at
each object's center pixel.  The label placement uses the same coordinate
transformation as the scene renderer:
\begin{align}
    p_x^{(i)} &= m_x + \frac{x_i}{100}\,(w - 2m_x), \\
    p_y^{(i)} &= (h - m_y) - \frac{y_i}{100}\,(h - 2m_y),
\end{align}
where $(x_i, y_i) \in [0,100]^2$ are the object's stored coordinates in
bottom-left origin space, $(w, h)$ are the rendered image dimensions, and
$(m_x, m_y)$ are pixel margins.  This formula maps logical canvas space
exactly to pixel space, so each label lands visually on top of its object.
Labels are rendered in black with a white outline for legibility on any background.

No feature text (color, shape, size, location) is passed to the listener.
The model must \emph{identify} object properties from its own visual perception,
\emph{match} them against the utterance, and \emph{output} a posterior over
the integer indices.

Figure~\ref{fig:io-pipeline} illustrates the full IO listener pipeline.

\paragraph{Posterior derivation from the model's response.}
Listener models may produce probabilities in several formats depending on the
prompting strategy used.  We implement three variants, each corresponding to
a distinct reasoning philosophy:

\begin{enumerate}
\item \textbf{io-direct} --- The model is instructed to output a JSON array of
      $N$ floats summing to 1.0, one per indexed object:
      \texttt{[p0, p1, \ldots]}.
      The prompt specifies only that probabilities must sum to 1; no reasoning
      structure is imposed.  The model directly encodes its confidence about
      each object index.  We parse the first JSON array matching
      \texttt{[\textbackslash d.,\textbackslash s]+} via regex and normalise.

\item \textbf{io-cot} (chain-of-thought) --- The model follows a structured
      three-step protocol: (1)~\textsc{observe} --- describe each object by
      index as seen in the image; (2)~\textsc{match} --- score each object
      0--10 based on utterance fit; (3)~\textsc{assign} --- convert scores
      to probabilities.
      Output is tagged: \texttt{SCORES: [s0, \ldots]} and
      \texttt{PROBS: [p0, \ldots]}.
      Parsing prefers the \texttt{PROBS} tag; falls back to \texttt{SCORES}
      (renormalised) if absent.

\item \textbf{io-elimination} --- The model explicitly lists \textsc{ruled\_out}
      indices (probability forced to 0), then scores the remaining
      \textsc{candidates} proportionally.
      Output: \texttt{RULED\_OUT: [i, j, \ldots]},
      \texttt{CANDIDATES: [i, j, \ldots]},
      \texttt{PROBS: [p0, \ldots]}.
      The elimination strategy is most aggressive: it is designed for scenes
      where the utterance strongly contradicts most objects, so the posterior
      can concentrate on a small candidate set.
\end{enumerate}

All three strategies share the same index-annotated image; they differ only in
how they instruct the VLM to reason about the assignment.
In all cases the final posterior is renormalised to sum to 1 if numerical
errors are present; if parsing fails entirely, a uniform distribution
$[1/N, \ldots, 1/N]$ is used as a fallback.

\paragraph{From posterior to accuracy.}
Let $\mathbf{p} = (p_0, \ldots, p_{N-1})$ be the normalised posterior over
$N$ objects.  The listener's prediction is the argmax:
\begin{equation}
    \hat{t} = \arg\max_{i}\, p_i.
    \label{eq:argmax}
\end{equation}
Accuracy is the fraction of scenes where $\hat{t} = t^*$ (the ground-truth
target index).

For cost-aware clarification, the decision rule is:
\begin{equation}
    \text{ask} \iff \max_i p_i < 1 - c.
    \label{eq:eu-rule}
\end{equation}
When the listener \emph{asks} (abstains), it neither contributes a correct nor
an incorrect prediction; the scene counts only as a clarification cost $c$.
Cost-Penalized Accuracy (CPA) aggregates both outcomes:
\begin{equation}
    \text{CPA}(c) = \frac{|\{\text{correct commits}\}|}{n} - c \cdot \frac{|\{\text{asks}\}|}{n}
                 = \text{Acc} - c \cdot \text{Ask\%},
    \label{eq:cpa}
\end{equation}
where $n$ is the total number of scenes.  A listener that never asks ($\text{Ask\%}=0$)
maximises CPA iff its accuracy is high; one that asks frequently must have
correspondingly high accuracy on the scenes it does commit to, or it will
be penalised.

Table~\ref{tab:io-vs-ta} summarises what information each listener type receives.
Under the TA protocol, the object list effectively \emph{solves} the
reference problem as a text lookup: the model merely needs to find which row in
the list best matches the utterance.  Under the IO protocol, the model must
ground the utterance in visual perception --- identifying colors, shapes, and
spatial relationships from pixels.

\subsection{Analysis}

\paragraph{Finding 1: \textit{io-index} dominates all other IO listeners, with the gap widening at larger $N$.}
\textit{io-index} --- which outputs a single integer and never asks --- achieves
the highest CPA in every (speaker, dataset) condition, often by a substantial
margin (Table~\ref{tab:io-c05}).
At $N{=}6$ the advantage over \textit{io-direct} is modest ($\Delta\text{CPA}
\approx 0.10$--$0.14$); by $N{=}8$ it widens to $\approx 0.15$--$0.21$; at
$N{=}10$ it reaches $\approx 0.17$--$0.20$.

Two mechanisms explain this pattern.
First, \emph{posterior flatness grows with $N$}: with more candidate objects in a
small image, the model spreads probability across a larger set of plausible
matches, depressing $\max_i p_i$ even when the argmax is correct.
At $N{=}6$, $\max p \approx 0.65$ on average, so only $\sim 8$\% of scenes
fall below the $1{-}c = 0.5$ commit threshold.
At $N{=}10$, $\max p$ drops to $\approx 0.55$, pushing the ask rate to
$25$--$35$\% and costing 0.5 CPA points per ask.
\textit{io-index} bypasses this collapse entirely.

Second, \emph{prompt-induced uncertainty inflation}: when the model is asked for a
probability distribution it explicitly deliberates about its uncertainty and
encodes it in the numbers, often spreading mass over 2--3 candidates even when
its best guess is unambiguous.
When asked for a single integer it suppresses this deliberation and commits
directly to the argmax.
The argmax prediction is typically the same in both cases, but \textit{io-direct}
pays the EU ask penalty that \textit{io-index} avoids.

The implication is that the EU clarification mechanism as implemented
\emph{backfires} at larger scene sizes: the listener becomes so visually
uncertain that it asks on scenes it would have answered correctly, spending
clarification cost without accuracy benefit.
A well-calibrated EU rule should ask only when the listener is actually likely
to be wrong; here the posterior flatness is driven by visual perceptual
difficulty rather than genuine referential ambiguity in the utterance.
Addressing this requires either posterior recalibration (temperature scaling) or
a separate signal for utterance ambiguity versus perceptual uncertainty.

\paragraph{Finding 2: TA listeners substantially overestimate listener accuracy.}
Comparing Table~\ref{tab:io-c05} (IO) with Table~\ref{tab:fullscale} (TA),
the mean CPA of the best IO listener (\textit{io-direct}) is
$0.715$--$0.840$ across datasets, versus $0.900$--$0.997$ for TA listeners.
The drop of $\approx 0.20$ CPA points represents the contribution of textual
feature lookup to TA performance.
A naive TA listener could identify the correct object with high probability
by simple token matching, even without ever rendering the image; IO listeners
cannot resort to this shortcut.
This motivates using IO listeners as the \emph{scientifically honest} evaluation
protocol for all future experiments.

\paragraph{Finding 3: \textit{io-direct} is the best IO strategy that uses a probability distribution.}
Across all 72 (speaker, dataset, cost) conditions, \textit{io-direct} achieves
the highest CPA in every single case.
\textit{io-elimination} is consistently 5--15 CPA points below \textit{io-direct}
at $c{=}0.5$.
\textit{io-cot} is catastrophically bad: its CPA falls below zero at $c{=}0.10$
and $c{=}0.25$ for $N \geq 8$ datasets.

The failure mode of \textit{io-cot} is instructive.  When asked to explicitly
\emph{describe} each object before assigning a score, the model's uncertainty is
made salient to itself: it generates descriptions like ``Object 3 appears to be
either a blue or green circle'' and then assigns low scores to all objects because
it cannot resolve the visual ambiguity.  The resulting posterior is nearly uniform
(entropy $\approx \ln N$), triggering the EU rule to ask in $>60\%$ of scenes.
At $c{=}0.10$, even the 6\% saving on wrong answers cannot compensate for the
60\% clarification cost.

In contrast, \textit{io-direct} suppresses the model's expressed uncertainty by
asking for a direct probability assignment without an intermediate description
step.  The model implicitly commits to a visual interpretation and encodes its
confidence directly in the array, rather than propagating uncertainty through a
chain of reasoning steps.
This is analogous to the difference between fast (System 1) and slow (System 2)
reasoning: forcing explicit deliberation can \emph{increase} apparent uncertainty
even when the intuitive judgment is correct.

\paragraph{Finding 4: \textit{io-elimination} becomes overconservative at scale.}
The elimination strategy is designed for scenes where the utterance strongly
rules out most objects.  Its ask rate grows monotonically with $N$: at
$N{=}6$ it asks $\approx 22\%$ of the time (CPA $\approx 0.69$); at $N{=}10$
it asks $\approx 40\%$ (CPA $\approx 0.55$).
As the candidate set grows, the model identifies fewer objects it is
confident it can \emph{visually} rule out, leaving a larger candidate set
with a flat posterior.
The direct-rank listener avoids this collapse by assigning probabilities in
one pass without an explicit rule-out step.

\paragraph{Finding 5: Speaker strategy matters less under IO evaluation.}
Under TA evaluation, \textit{canonical\_vllm} outperformed \textit{natural} by
$\Delta\text{CPA} \approx 0.025$ (Table~\ref{tab:fullscale}).
Under IO evaluation, the spread across speaker strategies is
$\Delta\text{CPA} \leq 0.012$ for \textit{io-direct} (Table~\ref{tab:io-c05}).
The speaker gap shrinks because the IO listener's principal bottleneck is
\emph{visual object recognition} (mapping pixels to referent), not utterance
parsing.  A more discriminative speaker utterance helps when the listener
can parse feature text efficiently; it helps much less when the listener
must visually ground the reference.
This result suggests that speaker engineering yields diminishing returns when
listeners operate under genuine visual uncertainty.

\paragraph{Finding 6: Performance degrades gracefully with $N$.}
For \textit{io-direct} at $c{=}0.5$, mean CPA (averaged over speakers and
\textit{none}/\textit{force}) decreases from $\approx 0.77$ at $N{=}6$ to
$\approx 0.72$ at $N{=}8$ to $\approx 0.70$ at $N{=}10$.
This $\approx 7\%$ absolute degradation reflects the genuine difficulty of
picking out one object from an increasingly cluttered scene --- a difficulty
that affects humans as well as VLMs.
The \textit{force} variant is consistently $2$--$5$ CPA points harder than
\textit{none} at the same $N$, confirming that forced same-color distractors
introduce real visual ambiguity rather than merely making scenes larger.

\paragraph{Finding 7: The residual gap is dominated by listener capacity, not speaker quality.}
The best IO result (\textit{feature-canonical} + \textit{io-direct} at $N{=}6$,
\textit{force}: CPA $= 0.840$) is compared with the oracle ceiling
(\textit{feature-canonical} + TA \textit{direct-rank}: CPA $= 0.995$).
The $0.155$ gap is attributable almost entirely to listener capacity:
the oracle produces maximally discriminating utterances, yet the IO listener
still lags by 15 CPA points.
This motivates future work on listener pretraining or calibration rather than
further speaker engineering.

The key takeaway for the broader cost-aware clarification framework is that
\textbf{the source of uncertainty matters}.
Under TA evaluation, ambiguity arises from utterance underspecification
(the speaker failed to mention a distinguishing feature);
the posterior is flat because multiple objects match the text.
Under IO evaluation, ambiguity arises from \emph{perceptual} uncertainty
(the listener cannot reliably identify an object's color or shape from
a $330{\times}328$ pixel image with up to ten small shapes);
the posterior is flat even when the utterance is perfectly discriminative.
These are qualitatively different failure modes and call for different
remedies: speaker engineering targets the former; listener calibration
and higher-resolution rendering target the latter.
The cost-utility clarification rule (Eq.~\ref{eq:eu-rule}) correctly
triggers asking in both cases, but the downstream interventions required
to reduce the ask rate differ accordingly.

\textbf{Q: How is referential entropy computed?}
$H(T \mid u) = -\sum_{i=1}^{N} P(t_i \mid u)\log P(t_i \mid u)$,
where $P(t_i \mid u)$ is the posterior over candidate objects obtained via LLM
log-likelihood scoring.  High entropy ($\approx \ln N$) indicates a uniform posterior
(fully ambiguous utterance); entropy near zero indicates a nearly unique referent.

\textbf{Q: Over what are Brier score and ECE computed?}
Both are computed over the $N$-dimensional predicted distribution $P(t \mid u)$ per
instance (see Section~2).  Brier score uses the full distribution; ECE uses only the
top confidence per instance.  The ground truth is the one-hot label of the actual
target object.
